<objective>Replace native HTTP HLS download with ffmpeg subprocess, add mp4 remux after stream ends.</objective>

<context>
Current implementation: `src/hls.ts` fetches m3u8 playlist via HTTP, downloads segments concurrently, concatenates buffers, writes to `.ts` file.

Target: Use `ffmpeg -i <m3u8_url> -c copy output.ts` for download, then `ffmpeg -i output.ts -c copy output.mp4` for remux.

Dependencies: Add `nano-spawn` (zero-dep, 4.8KB, Promise-based, AbortSignal support).

Key files:
- `src/hls.ts` — HLS download (rewrite with ffmpeg)
- `src/recorder.ts` — Recorder class (add remux after recording stops)
- `src/types.ts` — DownloadOptions, RecordOptions (add remux option)
- `src/errors.ts` — Error classes (add FfmpegError)
- `src/index.ts` — Public exports
- `bin/bigorec.ts` — CLI commands
</context>

<acceptance_criteria>

## AC-1
Given ffmpeg is installed, When `downloadHls()` is called, Then ffmpeg process is spawned to download HLS stream to .ts file and returns output path.

## AC-2
Given a recording completes or is stopped, When recording finishes, Then .ts file is remuxed to .mp4 via ffmpeg and .ts is removed.

## AC-3
Given ffmpeg is not installed, When any ffmpeg operation is attempted, Then a clear FfmpegError is thrown with installation instructions.

</acceptance_criteria>
